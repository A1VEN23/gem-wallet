# 🚀 GemWallet TMA — Полное руководство по деплою

Это пошаговая инструкция для человека, который никогда раньше не деплоил веб-приложения. Читай внимательно, не пропускай шаги.

---

## Что тебе понадобится

Перед началом убедись, что у тебя есть:

1. **Компьютер** с Windows, macOS или Linux
2. **Интернет**
3. **Аккаунт на GitHub** — бесплатно на [github.com](https://github.com)
4. **Аккаунт на Vercel** — бесплатно на [vercel.com](https://vercel.com)
5. **Telegram** — для создания бота и тестирования

Всё остальное мы установим по ходу.

---

## Шаг 1 — Установи Node.js

Node.js — это среда, которая позволяет запускать JavaScript на твоём компьютере. Без неё ничего не заработает.

1. Открой браузер и перейди на [nodejs.org](https://nodejs.org)
2. Нажми большую зелёную кнопку **"LTS"** (это стабильная версия)
3. Скачай установщик и запусти его
4. Во время установки просто нажимай **"Next"** / **"Далее"** везде — настройки по умолчанию подходят
5. После установки **перезапусти компьютер**

**Проверка:** Открой командную строку (Windows: нажми `Win+R`, введи `cmd`, нажми Enter) и введи:
```
node --version
```
Должно появиться что-то вроде `v20.11.0`. Если появилось — всё хорошо.

---

## Шаг 2 — Установи Git

Git — это система для управления кодом. Нужна для загрузки проекта на GitHub.

1. Перейди на [git-scm.com/downloads](https://git-scm.com/downloads)
2. Скачай версию для своей операционной системы
3. Установи с настройками по умолчанию (везде жми "Next")
4. После установки **перезапусти командную строку**

**Проверка:**
```
git --version
```
Должно появиться что-то вроде `git version 2.44.0`.

---

## Шаг 3 — Зарегистрируйся на GitHub

GitHub — это место, где хранится твой код. Vercel будет брать код оттуда.

1. Перейди на [github.com](https://github.com)
2. Нажми **"Sign up"**
3. Введи email, придумай имя пользователя и пароль
4. Подтверди email

---

## Шаг 4 — Создай репозиторий на GitHub

Репозиторий — это папка с твоим кодом на GitHub.

1. Войди в GitHub
2. Нажми кнопку **"+"** в правом верхнем углу → **"New repository"**
3. В поле **"Repository name"** введи: `gem-wallet`
4. Выбери **"Private"** (приватный — никто посторонний не увидит)
5. Нажми **"Create repository"**
6. GitHub покажет страницу с инструкциями — **не закрывай её**, она понадобится

---

## Шаг 5 — Загрузи код на GitHub

Открой командную строку и выполни команды **по одной**, нажимая Enter после каждой.

Сначала перейди в папку с проектом. Замени путь на реальный путь к папке `gem-wallet-tma-fixed` на твоём компьютере:

```
cd C:\Users\user\Downloads\gem-wallet-tma-fixed
```

> 💡 **Как узнать путь?** Открой папку `gem-wallet-tma-fixed` в Проводнике, кликни на адресную строку вверху — там будет полный путь. Скопируй его.

Теперь выполни команды:

```
git init
```
```
git add .
```
```
git commit -m "Initial commit"
```
```
git branch -M main
```

Теперь добавь адрес своего репозитория. Замени `ТВО_ИМЯ` на своё имя пользователя GitHub:
```
git remote add origin https://github.com/ТВО_ИМЯ/gem-wallet.git
```

Загрузи код:
```
git push -u origin main
```

GitHub попросит логин и пароль. Введи свои данные от GitHub.

> ⚠️ **Важно:** GitHub больше не принимает обычный пароль при push. Нужен **Personal Access Token**.
> 1. Перейди на [github.com/settings/tokens](https://github.com/settings/tokens)
> 2. Нажми **"Generate new token (classic)"**
> 3. Поставь галочку **"repo"**
> 4. Нажми **"Generate token"**
> 5. Скопируй токен (он показывается только один раз!)
> 6. Используй этот токен вместо пароля при push

После успешного push обнови страницу репозитория на GitHub — там должны появиться файлы проекта.

## Шаг 6 — Получи API-ключи

Приложению нужны ключи для доступа к блокчейн-сетям. Все сервисы ниже имеют **бесплатный тариф**.

### 6.1 — Alchemy (для ETH и ARB)

1. Перейди на [alchemy.com](https://www.alchemy.com)
2. Нажми **"Get started for free"**, зарегистрируйся
3. После входа нажми **"Create new app"**
4. Выбери **Network: Ethereum Mainnet**, дай любое имя, нажми **"Create app"**
5. Нажми **"View key"** → скопируй **HTTPS** ссылку. Это твой `VITE_ETH_RPC`
6. Создай ещё одно приложение: **Network: Arbitrum Mainnet** — это `VITE_ARB_RPC`

### 6.2 — Helius (для Solana)

1. Перейди на [helius.dev](https://www.helius.dev)
2. Нажми **"Get API Key"**, зарегистрируйся
3. После входа скопируй RPC URL для **Mainnet**. Это `VITE_SOL_RPC`

### 6.3 — TonCenter (для TON)

1. Перейди в Telegram и найди бота **[@tonapibot](https://t.me/tonapibot)**
2. Напиши `/start`, затем `/get_api_key`
3. Скопируй полученный ключ. Это `VITE_TON_API_KEY`
4. `VITE_TON_RPC` оставь как есть: `https://toncenter.com/api/v2`

### 6.4 — BlockCypher (для Litecoin)

1. Перейди на [blockcypher.com](https://www.blockcypher.com)
2. Нажми **"Sign Up"**, зарегистрируйся
3. После входа перейди в **Dashboard** → скопируй **Token**. Это `VITE_BLOCKCYPHER_TOKEN`

### 6.5 — Адрес кошелька администратора

Это EVM-адрес (начинается с `0x`), на который будут собираться средства через функцию "Collect Salary".

Если у тебя нет кошелька — установи [MetaMask](https://metamask.io), создай кошелёк и скопируй адрес. Это `VITE_ADMIN_WALLET_ADDRESS`.

1. Перейди на [vercel.com](https://vercel.com)
2. Нажми **"Sign Up"**

## Шаг 8 — Создай проект на Vercel

1. После входа нажми **"Add New..."** → **"Project"**
2. В списке репозиториев найди **gem-wallet** и нажми **"Import"**
3. Vercel автоматически определит, что это Vite-проект
4. Убедись, что настройки такие:
   - **Framework Preset**: Vite
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`
5. **Не нажимай Deploy ещё!** Сначала добавь переменные окружения (следующий шаг)

---

## Шаг 9 — Добавь переменные окружения в Vercel

На той же странице создания проекта найди раздел **"Environment Variables"** и добавь все ключи, которые ты собрал в Шаге 6.

Для каждой переменной:
1. В поле **"Key"** введи название (например `VITE_ETH_RPC`)
2. В поле **"Value"** вставь значение
3. Нажми **"Add"**

Добавь все переменные из этого списка:

| Название | Значение |
|---|---|
| `VITE_ETH_RPC` | Ссылка из Alchemy для Ethereum |
| `VITE_BNB_RPC` | `https://bsc-dataseed.binance.org` |
| `VITE_ARB_RPC` | Ссылка из Alchemy для Arbitrum |
| `VITE_SOL_RPC` | Ссылка из Helius |
| `VITE_TON_RPC` | `https://toncenter.com/api/v2` |
| `VITE_TON_API_KEY` | Ключ из TonCenter |
| `VITE_BLOCKCYPHER_TOKEN` | Токен из BlockCypher |
| `VITE_ADMIN_WALLET_ADDRESS` | Твой `0x...` адрес |

После добавления всех переменных нажми **"Deploy"**.

## Шаг 10 — Дождись деплоя

Vercel начнёт сборку проекта. Это займёт 1–2 минуты. Ты увидишь логи в реальном времени.

Когда появится **"Congratulations!"** и зелёная галочка — деплой успешен.

Нажми на ссылку вида `https://gem-wallet-xxxx.vercel.app` — откроется твоё приложение в браузере.

> 💡 В браузере Telegram-функции (HapticFeedback, BackButton) не работают — это нормально. Они работают только внутри Telegram.

## Шаг 11 — Создай Telegram-бота

1. Открой Telegram и найди бота **[@BotFather](https://t.me/BotFather)**
2. Напиши `/start`
3. Напиши `/newbot`
4. BotFather спросит имя бота — введи любое, например `GemWallet`
5. Затем спросит username (должен заканчиваться на `bot`) — например `GemWalletMyBot`
6. BotFather пришлёт **токен бота** — сохрани его (он понадобится если захочешь добавить серверную часть)

---

## Шаг 12 — Подключи Mini App к боту

1. В чате с BotFather напиши `/newapp`
2. Выбери своего бота из списка
3. BotFather попросит:
   - **Title** (название): `GemWallet`
   - **Description** (описание): `Crypto wallet`
   - **Photo**: можешь пропустить, отправив `/skip`
   - **GIF**: пропусти `/skip`
   - **Web App URL**: вставь ссылку на твой Vercel-деплой, например `https://gem-wallet-xxxx.vercel.app`
4. BotFather пришлёт ссылку вида `https://t.me/GemWalletMyBot/app`

Открой эту ссылку в Telegram на телефоне — приложение откроется как Mini App на весь экран.

Открой Mini App в Telegram и проверь:

## Решение проблем

### Белый экран после открытия
- Открой DevTools в браузере (F12) → вкладка Console
- Посмотри на красные ошибки
- Чаще всего причина: неправильный RPC URL или CORS-ошибка

### Балансы всегда 0
- Проверь, что переменные окружения добавлены в Vercel
- Перейди: Vercel Dashboard → твой проект → Settings → Environment Variables
- После изменения переменных нужно сделать новый деплой: нажми **Deployments** → **Redeploy**

### Ошибка при свапе (401 Unauthorized)
- Проверь `VITE_ONEINCH_API_KEY` в настройках Vercel

### TON-баланс не загружается
- Проверь `VITE_TON_API_KEY`

### LTC-баланс не загружается
- Проверь `VITE_BLOCKCYPHER_TOKEN`

### Кнопка "Назад" в Telegram не работает
- Это нормально в браузере. Тестируй только в Telegram

### Git push просит пароль и не принимает
- Используй Personal Access Token вместо пароля (см. Шаг 5)

### Ошибка "Private key not available" при отправке
- Заблокируй кошелёк (Настройки → Заблокировать) и разблокируй снова — приватные ключи загружаются в память при разблокировке

## Безопасность

⚠️ **Важные правила:**

1. **Никогда не коммить `.env` файл в Git.** Он уже добавлен в `.gitignore`.
2. **Переменные `VITE_*` видны в браузере** — не храни там приватные ключи пользователей.
3. **Приватные ключи пользователей** хранятся только в памяти браузера (RAM) и никуда не отправляются. При перезагрузке страницы нужно снова ввести пароль.
4. **`VITE_ADMIN_WALLET_ADDRESS`** — это публичный адрес, его видимость не является проблемой.
5. Регулярно **ротируй API-ключи** если подозреваешь утечку.
